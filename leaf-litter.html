Abstract

A challenge operating system, not just another agent.

Capture the Flag competitions are a rotating stack of completely different disciplines forced into the same clock. A reversing binary, a kernel pwn target, a JWT bug, a lattice attack, a damaged QR code, a memory dump, and a weird mixed-category puzzle do not want the same workflow. Leaf Litter starts from that reality instead of flattening everything into one giant prompt.

Can you build a system that behaves like a serious CTF operator?
  • Ingest messy challenge inputs: files, zips, copied descriptions, service metadata, and half-clean workspaces without forcing manual cleanup first.
  • Choose the right workflow: decide what kind of problem it is, which tools belong, and when a cheap first pass is enough.
  • Operate like infrastructure: recover from crashes, remember prior work, benchmark honestly, and leave behind reports, snapshots, and artifacts.

Leaf Litter began as an answer to the design philosophy behind Squid Agent and quickly grew into something larger: a challenge normalizer, capability detector, fast-path Codex wrapper, generated category-specific multi-agent runtime, control center, benchmark harness, replay engine, memory layer, and reporting stack.

Lineage

Why Squid Agent mattered, and where Leaf Litter pushes further.

What made Squid Agent compelling was not just that it could solve CTF problems. It treated orchestration, specialization, and benchmarking like real system concerns. That framing clicked immediately: if strong human teams specialize by category, an automated system should not avoid specialization on purpose.

  • Category specialization: reversing, pwn, crypto, web, forensics, and misc should have different role graphs and different attack postures.
  • Tool discipline: tool access is part of agent design, not a loose implementation detail.
  • Benchmark-first thinking: the project should be measured like engineering work, not narrated like a demo.

Leaf Litter follows that line directly, then leans harder into operator experience, environment readiness, fallback behavior, live platform automation, persistence, replayability, runtime safety, and post-mortem visibility.

Architecture

Five layers tied into one coordinated stack.

Challenge Input
  -> Normalizer
  -> Normalized Descriptor + Files Workspace
  -> Capability Detection + Preflight + Solve-Time Estimation
  -> Codex-First Fast Path
  -> Category-Specific Preanalysis
  -> Generated Multi-Agent Runtime
  -> Strategy Ensemble / Replay / Escalation / Fallback
  -> Flag Resolution + Validation + Optional Platform Submission
  -> Memory / RAG / Snapshots / Reports / Logs / Artifacts
  • Challenge intake and normalization: standardize messy input into stable descriptors and a consistent file workspace.
  • Fast-path and category-aware solving: take an inexpensive first shot, then escalate into generated specialist runtimes when the challenge deserves it.
  • Self-healing runtime control: watchdogs, budget controls, retries, and model routing keep the system from wasting time or money blindly.
  • Operator-facing control surfaces: terminal and browser interfaces make the solver feel usable under pressure.
  • Memory, platform mode, and replay: every run feeds reports, retrieval, and reproducible artifacts instead of disappearing after the flag.

The layers are meant to behave like one stack. The normalizer feeds the generator, the generator feeds the solver, and the solver feeds memory, RAG, snapshots, and reports.

Solve Pipeline

Normalize first. Generate the solver. Take the cheap win if it is obvious.

CTF inputs rarely arrive in a clean benchmark format. Leaf Litter treats ingestion as a first-class system concern: it extracts artifacts into a stable workspace, infers likely entry points, writes normalized metadata, and gives the rest of the stack something reliable to work with.

  • Generated solvers: prompts, curated tool lists, role graphs, model assignments, step caps, budget wiring, and validation hooks are created from the normalized challenge instead of hard-coded once.
  • Role-specific tool access: a pwn debugger should not see the same tool surface as a crypto critic or a web exploit developer.
  • Codex-first fast path: easy challenges get one disciplined, low-cost direct shot before the full runtime spins up.

That combination keeps obvious wins cheap while preserving a much deeper orchestration path for the hard problems.

Specialist Tracks

Specialization beats fake universality.

Leaf Litter does not pretend rev, pwn, crypto, web, and forensics are the same problem with different labels. Each track gets its own role graph, tool surface, preanalysis, and strategy profile.

  • Reverse engineering: anti-analysis detection, packer handling, WASM workflows, unpacking, and symbolic-analysis hints.
  • Pwn: mitigation detection, exploit-primitive inference, heap shaping, ROP construction, seccomp-aware fallbacks, container escape, and kernel-specific paths.
  • Crypto: source-aware probing for lattice attacks, RSA variants, PRNG failures, ECB behavior, and oracle-style bugs with a disciplined write-run-interpret-review cycle.
  • Web: fingerprinting, source-map extraction, bundle analysis, injection discovery, request smuggling checks, and stateful exploit sequencing across multi-step chains.
  • Forensics and misc: phased orchestration for images, audio, video, pcaps, disk images, memory dumps, stego, encodings, and bridge paths when a misc challenge is really a hidden specialist problem.
Runtime Control

The runtime is built to heal, rotate, and keep evidence.

Agents stall. APIs timeout. Context windows overflow. Tool calls crash. Exploit attempts work locally and fail remotely. Leaf Litter treats recovery as part of the design, not something to patch in later.

  • Watchdog recovery: detect rate limits, timeouts, JSON parse errors, subprocess crashes, and tool failures, then adapt with backoff, chunking, forced JSON, or tool skips.
  • Budget governance: track token usage, challenge cost, global cost, warnings, pause thresholds, and escalation into stronger models when cheap ones stop making progress.
  • Strategy systems: rotate through balanced, aggressive, and conservative profiles while excluding already-failed approaches.
  • Partial flag assembly: store fragments from tool outputs and agent responses, then attempt reconstruction later when the final string only appears in pieces.
  • Human-in-the-loop alerts: allow a human to inject a hint or approve continued spending without manually taking over the entire run.
Operations

Memory, control surfaces, live platform mode, and reproducible reports.

Leaf Litter is designed to feel like an operating system for challenge work, not a one-off benchmark script. That shows up in how it remembers, how it is driven, and what it leaves behind after a run.

  • Memory and RAG: session memory, FAISS-backed category stores, artifact indexing, solve-time estimation, and retrieval-guided strategies keep the system from starting cold.
  • Control plane: a lightweight terminal UI for health checks and guided launches, plus a browser control center for file intake, metadata, logs, previews, and process monitoring.
  • Benchmark mode: isolated workspaces, parallel challenge execution, and JSON or Markdown reporting turn the project into something measurable.
  • Platform mode: CTFd login, queue discovery, attachment download, solver generation, optional auto-submission, leaderboard awareness, and submission backoff connect local workflows to live competition play.
  • Trust and reproducibility: flag validation, consensus options, snapshots, command traces, post-mortems, and replayable artifacts make solved and failed runs equally useful.
Next Orbit

Why this matters, and where the project goes next.

Leaf Litter matters because it points toward a better question: what happens when CTF automation stops being a novelty and starts behaving like infrastructure? The interesting part is not that it can solve some challenges. The interesting part is that it knows how to load them, plan them, adapt during them, recover from failure, remember what happened, and scale from a single local file to a live competitive queue.

  • Deeper category stacks: stronger large-binary reversing, sharper heap and kernel pwn automation, tighter crypto reformulation, and more reliable web exploit confirmation.
  • Stronger learning loops: better retrieval scoring, smarter promotion and pruning, and more useful solve-template libraries.
  • Large-target handling: context partitioning, longer-horizon state retention, hierarchical tasking, and cleaner subproblem decomposition.
  • Sharper live competition behavior: better queue triage, smarter first-blood prioritization, and stronger submission consensus.
  • Better tooling and evaluation: deeper reversing, symbolic, fuzzing, and exploit-verification infrastructure with clearer regression tracking.
  • Human-agent collaboration: better instincts for when to ask for a hint, approval, or a category pivot instead of blindly continuing.
Squid Agent inspired the architecture. Leaf Litter is where that architecture starts turning into a broader operational system.
